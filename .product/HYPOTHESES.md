# Hypotheses

## [2026-04-19] Non-Telegram frontend for ingestion
**State:** speculative
**Statement:** A lightweight web UI for pasting articles and links will be built alongside the Telegram bot.
**Resolves when:** User builds and uses it; decides whether it stays or Telegram covers all ingestion.
**Shapes:** Whether ingestion is Telegram-exclusive or multi-surface in Phase 1.

## [2026-04-19] On-demand UX will emerge from use
**State:** needs-feel
**Statement:** The right interaction pattern for on-demand sessions (command, menu, free text) isn't clear before first use.
**Resolves when:** One week of daily use in Phase 1.
**Shapes:** Topic menu and direct-text invocation design.

## [2026-04-19] Browser extension for knowledge capture
**State:** speculative
**Statement:** A browser extension (highlight → push to Telegram bot) will be built in Phase 3 for passive article capture.
**Resolves when:** Phase 1 and 2 are proven; browser extension becomes worth the maintenance overhead.
**Shapes:** Phase 3 scope and ingestion surface area.

## [2026-04-19] Model strategy: Gemini for discussion, local for quiz
**State:** belief
**Statement:** Gemini (1M+ context) handles doc ingestion and Socratic discussion; a local model handles quiz rephrasing; a smart API model handles /decide evaluation.
**Resolves when:** Phase 1 implementation — actual cost and quality tradeoffs become visible.
**Shapes:** Infrastructure setup, API spend, latency expectations.
